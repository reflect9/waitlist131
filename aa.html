One powerful aspect of static web generators is that the UI is nearly invisible -- it's just git. Committing a new file automatically generates a web page, updates sidebars and navigation menus, and so on.

When you connect your repository to your office suite, the UI becomes even more invisible -- completely imperceptible to nontechnical users. They only need to know that whatever they put into the right folder "magically" ends up on their web site.

Here's a walkthrough of that usage model in Google Drive using a beautiful designer portfolio that you can clone in Cloudstitch and try yourself.

The goal of this video is to show you what managing a website with Cloudstitch feels like in practice. And while this example uses a pre-configured Jekyll theme, the basic workflow you see is the same regardless of the template and static site generator.
